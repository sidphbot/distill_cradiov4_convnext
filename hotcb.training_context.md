## Model Architecture
- **Teacher**: NVIDIA C-RADIOv4-H (ViT-Huge, `nvidia/C-RADIOv4-H`), frozen, half-precision
- **Student**: ConvNeXt-Tiny (`convnext_tiny.dinov3_lvd1689m`), pretrained, features_only at indices (2, 3)
- **Parameters**: ~30.3M (student + heads)
- **Key layers**:
  - `SummaryHead`: Linear projection from student stage-3 channels (Cs) to teacher summary dim (Ct)
  - `SpatialHead`: 1x1 Conv projection from student stage-2 channels (Cs_sp) to teacher spatial dim (Dt), with bilinear upsampling to teacher grid (Ht x Wt) and optional norm capping (spatial_norm_cap=50.0)
- Input size: 416x416, patch_size=16 -> teacher grid 26x26

## Loss Function
- **Type**: Multi-task weighted sum with 6 terms + consistency
- **Terms** (all tunable via hotcb `mutable_state`):

| Term | Weight/Ramp | Default | Description |
|------|-------------|---------|-------------|
| `lambda_summary` | static | 1.0 | Cosine loss on summary (CLS-like) features |
| `lambda_spatial` | static | 1.0 | Cosine loss on spatial token features |
| `lambda_mse` | static | 0.25 | MSE loss weight (applied to both summary and spatial) |
| `ramp_mse_sp_end` | ramp 0.5->1.5 | warmup 20% of epoch | Spatial MSE ramp endpoint |
| `ramp_grad_end` | ramp 0.1->1.3 | warmup 35% of epoch | Sobel gradient matching ramp endpoint |
| `ramp_cons_summary_end` | ramp 0.0->0.5 | warmup 15% of epoch | Consistency loss (summary) ramp endpoint |
| `ramp_cons_spatial_end` | ramp 0.0->0.5 | warmup 30% of epoch | Consistency loss (spatial) ramp endpoint |
| `aug_strength_end` | ramp 0.0->0.6 | warmup 25% of epoch | Augmentation blend strength |

- **Dual-pass training**: Each step runs clean input (distillation loss) then augmented input (consistency loss). Consistency encourages augmentation invariance in student features.
- **Key metric relationship**:
  - `cos_sum` and `cos_sp` directly drive `alignment_score` (0.30 weight each)
  - `hf_cos` (high-frequency cosine) adds 0.20 weight
  - `act_f1` (top-k activation F1) adds 0.30 weight
  - `hf_mse` penalizes with -0.10 weight
  - Overall alignment = 0.25 * oi_val + 0.75 * avg(off-dist sources)

## Optimizer & Scheduler
- **Optimizer**: AdamW, 6 param groups (separate norm/bias/weight groups per module)
  - lr=1.5e-4, wd=0.08, betas=(0.9, 0.999), eps=1e-8
- **Scheduler**: None (flat lr, no decay)
- **Gradient clipping**: 1.0 (manual, after unscale)
- **Grad accumulation**: 4 micro-steps
- **Known sensitivities**: With no scheduler, lr is the primary optimization lever. The flat rate means any lr change via hotcb persists for the rest of training.

## Data Pipeline
- **Training data**: Open Images v7 (`/media/burplord/kraken_data/oidv7`), 800k images cap
- **Validation sources** (1000 images each):
  - `oi_val`: 2% holdout from OI (in-distribution)
  - `coco_train`: COCO train split (off-distribution)
  - `imagenet_train`: ImageNet train split (off-distribution)
- **Online loading**: `OnlineCollate` loads/resizes at batch time, 50/50 pad-or-squash per (epoch, path) hash
- **Augmentations** (alpha-blended by ramping strength):
  - RandomBrightnessContrast (p=0.5)
  - HueSaturationValue (p=0.4)
  - GaussianBlur (p=0.3)
  - GaussNoise (p=0.3)
- **Batch size**: 48, num_workers=8, persistent_workers, prefetch_factor=4

## Notable Code Patterns
- **Manual optimization**: `automatic_optimization=False` for explicit gradient accumulation and immediate graph cleanup (memory-critical with ViT-Huge teacher)
- **Teacher stored via `object.__setattr__`**: Prevents Lightning from managing teacher params (must stay eval/frozen)
- **Mixed precision**: AMP autocast for both student passes, GradScaler for accumulation
- **Checkpoint strategy**: Saves on val loss improvement (oi_val source)
- **`mutable_state` dict**: 8 tunable weights exposed to hotcb, read every training step from `self.mutable_state["weights"]`
- **Memory management**: Explicit `del` and `torch.cuda.empty_cache()` between teacher/student forward passes
- **Steps per epoch**: ~16,667 (800k / 48), but hotcb runs with max_steps=800
